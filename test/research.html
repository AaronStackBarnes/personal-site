Advanced Penetration Testing Techniques

SQL Injection Deep Dive

SQL injection remains one of the most critical web vulnerabilities. The basic technique involves inserting malicious SQL code into application queries. For example, entering ' OR '1'='1 into a login field can bypass authentication entirely. More advanced techniques include UNION-based injection to extract database contents, blind SQL injection using time delays, and second-order injection where payloads are stored and executed later.

Password Cracking Methodologies

Modern password cracking combines multiple approaches. Dictionary attacks use wordlists like rockyou.txt. Rule-based attacks apply transformations (adding numbers, leetspeak). Hashcat and John the Ripper support GPU acceleration for faster cracking. Rainbow tables provide pre-computed hash lookups. For bcrypt and Argon2, attackers must rely on slower methods due to computational costs.

Network Exploitation

Network penetration often begins with reconnaissance using nmap for port scanning and service enumeration. Metasploit provides exploit modules for known vulnerabilities. Man-in-the-middle attacks using tools like Bettercap can intercept credentials on local networks. ARP spoofing redirects traffic through attacker-controlled systems.

Privilege Escalation

After initial access, attackers seek higher privileges. On Linux, SUID binaries, kernel exploits, and misconfigured sudo rules are common vectors. Windows privilege escalation often exploits service misconfigurations, unquoted service paths, or token impersonation. Tools like LinPEAS and WinPEAS automate enumeration of escalation paths.
